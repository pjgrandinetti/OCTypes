Detailed Description

Implements an autorelease pool for OCTypes.

Author: Philip
Date: May 10, 2025

OCAutoreleasePool provides a mechanism for managing OCType objects using a delayed‚Äêrelease pool. Usage:

  • Create pool: OCAutoreleasePoolCreate()
  • Autorelease: OCAutorelease(object)
  • Release pool: OCAutoreleasePoolRelease(pool)

Pools can be nested; releasing a nested pool drains only that pool.

◆ OCAutorelease()

const void * OCAutorelease ( const void * ptr)

Adds an object to the current (top-most) autorelease pool.

The object will receive a release message when the autorelease pool is released. The object must be a valid OCType-compatible pointer.

Parameters
ptrA pointer to the OCType object to be autoreleased.
Returns
The same pointer ptr that was passed in, allowing for chained calls.
Warning
If ptr is NULL, this may lead to issues when the autorelease pool is deallocated. Ensure ptr is non-NULL.

◆ OCAutoreleasePoolDrain()

void OCAutoreleasePoolDrain ( OCAutoreleasePoolRef pool)

Drains an autorelease pool, releasing all objects it contains without deallocating the pool itself. This is useful when the pool needs to be reused.

Parameters
poolThe OCAutoreleasePoolRef to be drained. Must not be NULL.

◆ OCAutoreleasePoolRelease()

bool OCAutoreleasePoolRelease ( OCAutoreleasePoolRef pool)

Releases an autorelease pool and all objects it contains.

When an autorelease pool is released, it sends a release message to all objects that were added to it.

Parameters
poolThe OCAutoreleasePoolRef to be released. Must not be NULL. If pool is not the most recently created pool, releasing it will also release all pools created after it.
Returns
true if the pool was successfully released, false otherwise.
